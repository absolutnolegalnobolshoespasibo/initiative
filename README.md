# Боевая инициатива — расширение для Owlbear Rodeo

Расширение добавляет в Owlbear Rodeo трекер инициативы для боя:
индивидуальная "изначальная инициатива" для каждого персонажа, таймер
готовности с угасанием инициативы, сортировка по инициативе с учётом
очерёдности готовности и автоматическое центрирование камеры по очереди
боя.

## Файлы

| Файл | Назначение |
|---|---|
| `manifest.json` | Манифест расширения — говорит Owlbear Rodeo, как его загрузить. |
| `index.html` | HTML-страница попап-окна (открывается по иконке слева). |
| `style.css` | Оформление (тёмная тема, в стиле интерфейса Owlbear Rodeo). |
| `main.js` | Связь с Owlbear Rodeo SDK, синхронизация состояния, отрисовка. |
| `logic.js` | Правила расчёта инициативы и порядка боя — без SDK и DOM. |
| `icon.svg` | Иконка действия (кубик) в левой панели. |
| `logic.test.mjs` | Необязательные тесты для `logic.js` (см. раздел "Тесты" в конце). |

Для самого расширения нужны только первые шесть файлов —
`logic.test.mjs` можно не загружать на хостинг, он не используется в
браузере.

Если захочешь разобраться или поменять правила боя — начни с
`logic.js`, там нет ничего, кроме математики и сортировки, с
комментариями на русском.

## Как реализованы правила

1. **Подготовка.** В попапе — список всех токенов слоя `Character` в
   порядке их добавления на сцену (по z-индексу токена), с полем
   "Изначальная инициатива" (целое число, по умолчанию 0). Кнопка
   "Начать раунд" видна всем, но активна только у мастера.

2. **Начать раунд.** У каждого персонажа инициатива становится
   `изначальная инициатива + 10`. Запускается таймер (отображается как
   ММ:СС с начала раунда). Кнопка "Начать раунд" превращается в "В
   бой", которая неактивна, пока не готовы все.

3. **Готовность.** У каждого персонажа появляется галочка. Если она
   стоит, инициатива уменьшается на 1 за каждые 30 секунд (значения
   `ROUND_BONUS = 10` и `DECAY_INTERVAL_MS = 30000` — в начале
   `logic.js`, их легко поменять). Повторное нажатие снимает галочку и
   возвращает инициативу к значению "после +10".

   - **Игроки** видят только персонажей из категории "доступны
     игрокам" (PC).
   - **Мастер** видит сначала персонажей "недоступных игрокам" (NPC),
     затем "доступных игрокам" (PC) — как и описано в задании.

   *Как определяется PC/NPC:* по умолчанию токен, созданный мастером,
   считается NPC, а созданный игроком — PC. В фазе подготовки мастер
   может вручную переключить метку **PC/NPC** у любого токена кнопкой
   рядом с полем инициативы — это сохраняется и используется дальше.

4. **В бой.** Когда все подтвердили готовность, мастер может нажать "В
   бой". Персонажи сортируются по текущей инициативе (по убыванию);
   при равенстве выше тот, кто поставил готовность раньше. Кнопка
   становится "Следующее существо", камера центрируется на первом в
   списке.

5. **Следующее существо.** Каждое нажатие центрирует камеру на
   следующем персонаже списка. После последнего — кнопка превращается
   в "Начать раунд", всё готово для нового раунда (изначальные
   инициативы сохраняются, угасание и готовность сбрасываются).

Дополнительно (этого не было в задании, но показалось полезным): в
фазах "Раунд" и "Бой" у мастера есть маленькая ссылка **"Сбросить
раунд / Сбросить бой"** — на случай ошибки, чтобы вернуться к
подготовке без необходимости проходить весь цикл.

## Установка

Owlbear Rodeo загружает расширения по публичной ссылке на
`manifest.json`, поэтому файлы нужно куда-то выложить. Проще и
бесплатнее всего — **GitHub Pages**.

### Шаг 1. Выложить файлы на GitHub Pages

1. Зайди на [github.com](https://github.com) и создай аккаунт, если
   его ещё нет.
2. Нажми **New repository**. Дай имя, например
   `owlbear-initiative-tracker`, выбери **Public**, создай репозиторий.
3. На странице репозитория нажми **Add file → Upload files** и
   перетащи туда все 6 файлов из этого комплекта (`manifest.json`,
   `index.html`, `style.css`, `main.js`, `logic.js`, `icon.svg`) —
   именно в корень репозитория, не в подпапку. Нажми **Commit
   changes**.
4. Перейди в **Settings → Pages**.
5. В разделе **Build and deployment** выбери **Source: Deploy from a
   branch**, ветку **main** и папку **/ (root)**, нажми **Save**.
6. Подожди минуту-две — GitHub покажет адрес вида
   `https://<твой-логин>.github.io/owlbear-initiative-tracker/`.
7. Ссылка на манифест будет:
   `https://<твой-логин>.github.io/owlbear-initiative-tracker/manifest.json`
   Открой её в браузере — должен показаться текст манифеста (JSON), а
   не ошибка 404.

### Шаг 2. Добавить расширение в Owlbear Rodeo

1. Открой [owlbear.rodeo](https://www.owlbear.rodeo) и зайди в свой
   аккаунт.
2. В меню профиля найди пункт **Add Extension** (Добавить
   расширение).
3. Вставь ссылку на `manifest.json` из шага 1 и подтверди.

### Шаг 3. Включить расширение в комнате

1. Открой (или создай) комнату.
2. Внизу слева открой меню **"..."** и выбери **Extensions**.
3. Включи **«Боевая инициатива»**.
4. В левой панели появится иконка-кубик — это и есть трекер
   инициативы. Открой её попап и начинай работу.

> ⚠️ Расширения видны и работают у всех, кто заходит в комнату.
> Игрокам стоит один раз сообщить, что в комнате включено сторонне
> расширение (это стандартная практика для Owlbear Rodeo).

### Если что-то не загружается

`main.js` подключает SDK Owlbear Rodeo через CDN:

```js
import OBR from "https://esm.sh/@owlbear-rodeo/sdk@3.1.0";
```

Если по какой-то причине `esm.sh` недоступен (блокировки сети), можно
заменить эту строку на:

```js
import OBR from "https://cdn.jsdelivr.net/npm/@owlbear-rodeo/sdk@3.1.0/+esm";
```

## Настройка

Все "числа правил" собраны в начале `logic.js`:

```js
export const ROUND_BONUS = 10;       // бонус при "Начать раунд"
export const DECAY_INTERVAL_MS = 30_000; // -1 инициативы за этот интервал готовности
```

Цвета и размеры — в начале `style.css`, в блоке `:root` (переменные
`--accent`, `--bg`, `--ready` и т. д.).

## Технические детали

- Состояние раунда/боя хранится в **метаданных комнаты** (видно всем,
  синхронизируется автоматически Owlbear Rodeo).
- "Изначальная инициатива" и метка PC/NPC хранятся в **метаданных
  каждого токена** — то есть остаются с персонажем между раундами и
  сценами.
- Персонажи — это токены на слое **Character** (`layer === "CHARACTER"`,
  `type === "IMAGE"`). Если токен не на этом слое, расширение его не
  увидит.

## Тесты (необязательно)

`logic.js` — это чистые функции без SDK и DOM, поэтому правила боя
(угасание инициативы, сортировка, переход между фазами) проверены
автоматическими тестами в `logic.test.mjs`. Чтобы запустить их, нужен
установленный [Node.js](https://nodejs.org) — затем в папке с файлами:

```bash
node --test
```

Все 19 проверок должны пройти (`pass 19`, `fail 0`). Для самого
расширения это не требуется — просто способ убедиться, что правила
считаются так, как описано выше, и безопасно менять константы
(`ROUND_BONUS`, `DECAY_INTERVAL_MS`) при необходимости.
